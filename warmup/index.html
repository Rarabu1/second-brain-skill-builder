Why bother

The terminal is just text in, text out.

It's a window into the same files and folders you already use. You type a command, the computer prints back a result. That's the whole thing.

It feels intimidating because it looks like the movies, but nothing you type below can affect your computer — this is a simulator that runs entirely in your browser. Once a command feels familiar here, the real one is identical.

Module 01

The sandbox

Type into the box below the terminal and press Enter. Try help any time to see what commands are available.

you@mac ~ ›

Exercise 1 — say hello

Type echo "hello world" and press Enter.

echo simply prints whatever you give it back to the screen. It's the universal "first command" in any new environment — a quick sanity check that the system is listening.

The quotes group the words together. You'd get the same result without them here, but quotes matter when your text has special characters.

Exercise 2 — who is logged in

Type whoami and press Enter.

This tells you which user the terminal is acting as. On your own laptop it's always you, but on a server or remote machine it's how you confirm who you're working as.

Exercise 3 — what's today

Type date and press Enter.

Prints the current date and time. Looks trivial, but it's a useful confirmation that the machine's clock matches what you expect — especially when files have wrong timestamps.

Exercise 4 — print the calendar

Type cal and press Enter.

This prints the current month as a calendar. A small, charming command that has shipped with Unix since the 1970s. Useful, occasionally; delightful, often.

Exercise 5 — where am I

Type pwd and press Enter.

pwd stands for "print working directory." Every terminal is always sitting inside some folder. This tells you which.

You should see a path starting with /Users/you. That's the simulator's home folder.

Exercise 6 — what's here

Type ls and press Enter.

ls lists what's in the current folder — the same thing you'd see if you opened it in Finder. Folders show in blue.

Exercise 7 — move into a folder

Type cd Documents, then ls.

cd means "change directory." Think of it as double-clicking a folder in Finder. The prompt updates to show your new location.

To go back up one level, type cd .. — two dots mean "the folder above this one."

Exercise 8 — jump home

Type cd ~ (or just cd) and press Enter.

The tilde ~ is shorthand for your home folder. From anywhere, this gets you back. Useful when you've wandered deep into a folder tree and want to start fresh.

Exercise 9 — find a file anywhere

Type find ~ -name "*.pdf" and press Enter.

This searches your entire home folder for files ending in .pdf. The * is a wildcard that matches anything. find is the single most useful command for "I know I have that file somewhere but I forget where."

Try a few variations: find ~ -name "*.jpg", or find ~/Downloads -name "*.pdf" to search only one folder.

Module 02

Now do it for real.

You've practiced the moves. Time to run them on your actual machine. Same commands, same outputs — just real files this time.

Step 1 — Open Terminal

Press ⌘ + Space, type Terminal, press Enter.

A window opens with a prompt that looks like the one above. You're now in your real home folder. Nothing here can hurt anything as long as you stick to the commands you've already practiced — they only read and look around, they never change or delete.

Step 2 — Run these, one by one

Copy each line, paste it into Terminal, press Enter.

echo "hello world"
whoami
date
cal
pwd
ls

Each one should feel familiar. The output is real this time — your actual username, today's actual date, the contents of your real home folder.

Step 3 — Find a file you actually own

The payoff exercise.

Try finding every PDF in your home folder. On most laptops, this finds a surprising number of them — tax forms, receipts, articles you saved and forgot about.

find ~ -name "*.pdf"

Swap the extension to look for other things: "*.jpg" for photos, "*.zip" for archives, "*resume*" to find anything with the word "resume" in its name.

find ~ -name "*resume*"

If you see permission errors scroll by, ignore them — they're folders the system protects. Your results are still in there.

That's it

You can use a terminal now.

Nine commands isn't everything, but it's a real foundation. echo, date, cal, whoami, pwd, ls, cd, find, and clear — that's most of what people actually type day to day.

Everything else builds from the same pattern: type the name of a tool, give it some inputs, press Enter, read the output. The commands you don't know yet work exactly the same way as the ones you do.

If you want to keep going, the next useful commands are mkdir (make a folder), cp (copy a file), mv (move or rename), and grep (search inside files). Same pattern, slightly more powerful.
